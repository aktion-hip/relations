Configuration

Identifier:
org.elbe.relations.configuration

Since:
0.9

Description:
This extension point allows to plug-in JDBC drivers.

Configuration Markup:

<!ELEMENT extension (Configuration)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT Configuration EMPTY>

<!ATTLIST Configuration

DBName              CDATA #REQUIRED

JDBCDriverClass     CDATA #REQUIRED

Subprotocol         CDATA #REQUIRED

embedded            (true | false)

Creator             CDATA #REQUIRED

canSetIdentityField (true | false) >

  • DBName - The name of the database the provided JDBC driver connects to.
  • JDBCDriverClass - Fully qualified name of database driver class provided by this plug-in's JDBC package.
  • Subprotocol - The database related part in the connection URL, e.g. jdbc:mysql://localhost/catalog.
  • embedded - Tells whether the database to be accessed is embedded (i.e. runs in the same Java thread as the application does) or not. For example: Derby is configured as embedded database by the Relations application. MySQL can be accessed as an external database.
  • Creator - Fully qualified name of the class that implements the org.elbe.relations.utility.IDBObjectCreator interface. This class can and should use org.elbe.relations.utility.AbstractDBObjectCreator as base class.
    The application describes the data model it uses in a RDMBS independent way by means of an XML file. The task of the Creator class is to provide an XSL stylesheet that transforms this XML into a set of CREATE TABLE and CREATE INDEX statements for the specific database.
  • canSetIdentityField - Some databases (e.g. MySQL) allow to set the identity field (primary key field) that is defined as auto_increment to a specified value whereas other (like Derby) don't allow this.

Examples:
A JDBC configuration looks like this. 
<plugin>

   
<extension
         point=
"org.elbe.relations.configuration"
>

      
<Configuration
            Creator=
"org.elbe.relations.mysql.MySQLCreator"

            DBName=
"MySQL"

            JDBCDriverClass=
"com.mysql.jdbc.Driver"

            Subprotocol=
"mysql"

            canSetIdentityField=
"true"

            embedded=
"false"
/>

   
</extension>


</plugin>

API Information:
To register a JDBC driver, fill in the driver class contained in the jar-file provided by the plug-in.
In addition, you have to implement a Creator class that implements the org.elbe.relations.utility.IDBObjectCreator interface. The main task of this class is to provide an XSL stylesheet (by implementing the IDBObjectCreator.getXSL() method that transforms the RDBMS independent specification of the application's data model into a database specific set of CREATE TABLE and CREATE INDEX statements.

Supplied Implementation:
The Relations application provides the org.elbe.relations.derby plug-in as default implementation of a JDBC configuration.

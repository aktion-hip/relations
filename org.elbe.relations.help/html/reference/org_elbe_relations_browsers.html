Relations Browsers

Identifier:
org.elbe.relations.browsers

Since:
0.9

Description:
Extension point for views (called relation or association browsers) to display an item's associations.

Configuration Markup:

<!ELEMENT extension (relationsBrowser)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT relationsBrowser EMPTY>

<!ATTLIST relationsBrowser

id    CDATA #REQUIRED

class CDATA #REQUIRED>

  • id - a unique name that will be used to identify this view (must be identical with org.eclipse.ui.views/view/id).
  • class - a fully qualified name of the class that implements org.elbe.relations.views.IRelationsBrowser (must be identical with org.eclipse.ui.views/view/class).

Examples:
The default browser's plugin.xml (see the org.elbe.relations.defaultBrowser plug-in) looks as follows: 
<plugin>

   
<extension
         point=
"org.elbe.relations.browsers"
>

      
<relationsBrowser
            class=
"org.elbe.relations.browser.RelationsDefaultBrowser"

            id=
"org.elbe.relations.views.RelationsDefaultView"
/>

   
</extension>

   
<extension
         point=
"org.eclipse.ui.views"
>

      
<view
            allowMultiple=
"false"

            class=
"org.elbe.relations.browser.RelationsDefaultBrowser"

            icon=
"icons/relations.gif"

            id=
"org.elbe.relations.views.RelationsDefaultView"

            name=
"Relations"
/>

   
</extension>


</plugin>

API Information:

Browser views must implement the org.elbe.relations.views.IRelationsBrowser interface. At the same time, they use the Eclipse view infrastructure. Thus they have to contribute to the org.eclipse.ui.views extension point and have to extend org.eclipse.ui.part.ViewPart.

Items displayed on the browser view must implement the org.elbe.relations.models.IBrowserItem interface.

Supplied Implementation:
The Relations application provides the org.elbe.relations.defaultBrowser plug-in as default implementation of a relations browser.

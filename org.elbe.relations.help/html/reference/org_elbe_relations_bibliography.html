Bibliography

Identifier:
org.elbe.relations.bibliography

Since:
0.9

Description:
This extension point allows to plug-in schemes to format bibliographical information (i.e. Text items) into a specified format for that it can be used in a bibliography.

Configuration Markup:

<!ELEMENT extension (bibliography)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT bibliography EMPTY>

<!ATTLIST bibliography

id               CDATA #REQUIRED

bibliographyName CDATA #REQUIRED

class            CDATA #REQUIRED>

  • id - a unique name that will be used to identify this bibliography schema
  • bibliographyName - name of the bibliography schema.
  • class - a fully qualified name of the class that implements org.elbe.relations.IBibliography.

Examples:
A bibliography scheme extension's plugin.xml may look as follows: 
<plugin>

   
<extension
         point=
"org.elbe.relations.bibliography"
>

      
<bibliography
            bibliographyName=
"MyBibliography (en)"

            class=
"org.xyz.biblio.MyBiblioEn"

            id=
"mybiblio.en"
/>

   
</extension>


</plugin>

API Information:
The value of the class attribute must be a fully qualified name of the class that implements org.elbe.relations.IBibliography. Developers of a bibliography plug-in may subclass org.elbe.relations.utility.AbstractBibliography.

Supplied Implementation:
An example of an extension for the Bibliography extension point is the org.elbe.relations.biblio.dft plug-in.

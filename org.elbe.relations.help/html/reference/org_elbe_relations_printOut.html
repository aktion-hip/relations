Relations PrintOut

Identifier:
org.elbe.relations.printOut

Since:
0.9

Description:
Extension point for plug-ins to print out selected relation items to a document.

Configuration Markup:

<!ELEMENT extension (printOut)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT printOut EMPTY>

<!ATTLIST printOut

id           CDATA #REQUIRED

printOutName CDATA #REQUIRED

fileType     CDATA #REQUIRED

fileTypeName CDATA #IMPLIED

class        CDATA #REQUIRED>

  • id - a unique name that will be used to identify this print out extension
  • printOutName - name of the print out.
  • fileType - file type of print out file, e.g. "*.txt". This file extension will be used in the file dialog to filter the files.
  • fileTypeName - name of the file type, used to describe the filter extension which the dialog will use. User-friendly short description shown for its corresponding filter.
  • class - a fully qualified name of the class that implements org.elbe.relations.print.IPrintOut.

Examples:
The default print out's plugin.xml (see the org.elbe.relations.print.dft plug-in) looks as follows: 
<plugin>

   
<extension
         point=
"org.elbe.relations.printOut"
>

      
<printOut
            class=
"org.elbe.relations.print.dft.PrintOut"

            fileType=
"*.txt"

            fileTypeName=
"Text file (.txt)"

            id=
"org.elbe.relations.print.dft.printOut"

            printOutName=
"Simple text file"
/>

   
</extension>


</plugin>

API Information:
Print out plug-ins must implement the org.elbe.relations.print.IPrintOut interface. They may make use of the functionality provided by org.elbe.relations.print.AbstractPrintOut by extending this abstract class.

Supplied Implementation:
The Relations application provides the org.elbe.relations.print.dft that prints out to a simple text file as default implementation of a print out plug-in.

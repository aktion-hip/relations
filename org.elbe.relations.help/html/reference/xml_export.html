The DTD of the XML export

The content of the Relations databases can be exported to an XML file (that can be compressed). XML is a future-proof format that makes it possible to process the content, e.g. by use of XSL transformation. This document shows the DTD and an example of an XML export.

Document Type Definition (DTD) of the exported XML

<!ELEMENT RelationsExport (TermEntries, TextEntries, PersonEntries, RelationEntries) >

<!ATTLIST RelationsExport date CDATA #REQUIRED >

<!ELEMENT TermEntries (TermEntry*) >

<!ELEMENT TextEntries (TextEntry*) >

<!ELEMENT PersonEntries (PersonEntry*) >

<!ELEMENT RelationEntries (RelationEntry*) >

<!ELEMENT TermEntry (ID, Title, Text, Created, Modified) >

<!ELEMENT TextEntry (ID, Type, Title, Subtitle, Author, CoAuthors, Year, Place, Publisher, Publication, Number, Volume, Pages, Text, Created, Modified) >

<!ELEMENT PersonEntry (ID, Name, Firstname, From, To, Created, Modified) >

<!ELEMENT RelationEntry (ID, Type1, Item1, Type2, Item2) >

<!ELEMENT ID (#PCDATA) >

<!ATTLIST ID field CDATA #REQUIRED
type (Number | String | Timestamp) #REQUIRED >

<!ELEMENT Title (#PCDATA) >

<!ATTLIST Title field CDATA #REQUIRED
type (Number | String | Timestamp) #REQUIRED >

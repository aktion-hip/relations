Workspace

The Relations application reserves file system space to store various application settings and make them available across multiple application sessions. If you use the default embedded database, this database will store all items you create in the workspace's data subdirectory.

By default, the application's workspace in the file system is a directory named workspace. This directory is created in the same directory you installed the Relations application.

However, you can change this setting and choose as application workspace any directory in the file system you want. To achieve this, you have to start the Relations application with the optional -data parameter, e.g.:

Relations.exe -data C:\Data\MyRelationsWorkspace

will use the directory named MyRelationsWorkspace in C:\Data as application workspace.

Note: If you use the embedded database to store the items and relations you create, be careful to not delete the application's workspace inadvertently. Deleting the workspace will delete all items you created. Furthermore, it may be a good idea to backup the embedded database from time to time.
